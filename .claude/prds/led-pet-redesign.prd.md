# LED Pet 第一性原理重设计

## Problem

当前 LED Pet 虚拟宠物系统将宠物视为游戏机制（喂食→累积→进化），而非情感连接的数字生命体。儿童无法与宠物建立真正的情感纽带，家长也缺乏有效的远程互动和引导工具。这种设计导致：

1. **情感连接缺失** — 宠物只是机械响应，没有"生命感"
2. **互动方式单一** — 只能喂食，无法建立情感纽带
3. **表达能力弱** — 宠物无法表达需求，儿童只能猜测
4. **成就感不足** — 进化只是数字累积，缺乏成长故事
5. **个性化缺失** — 所有宠物都一样，无法体现独特性

**成本**：儿童失去培养责任感和情感管理能力的机会，家长失去有效的教育工具。

## Evidence

- **Assumption** — 基于虚拟宠物设计第一性原理的推断，需要通过用户研究/原型测试验证
- **设计原则参考**：Tamagotchi、Digimon 等成功虚拟宠物的核心机制是情感连接，而非数值累积
- **儿童发展心理学**：4-10 岁儿童通过照顾行为发展同理心和责任感

## Users

- **Primary**: 儿童（3-12 岁，混合年龄段），通过物理按钮直接与宠物互动
  - **触发场景**：每天放学后、无聊时、想要奖励时
  - **核心需求**：情感连接、成就感、个性化表达
  - **交互方式**：设计适配最简交互（3-5岁），复杂度可选（9-12岁）
- **Secondary**: 家长，通过 Web 界面远程互动和控制
  - **触发场景**：想奖励孩子好行为、想了解孩子互动情况
  - **核心需求**：可见性、控制力、教育引导
- **Not for**: 成人用户、需要复杂游戏机制的用户

## Hypothesis

We believe **a need-based virtual pet system with personality development and branch evolution** will **create emotional attachment and meaningful growth** for **children aged 3-12**.

We'll know we're right when:
- 儿童在 2 周后能描述宠物的"性格"（如"它很爱玩"、"它喜欢睡觉"）
- 儿童每天至少主动互动 1 次（喂食、抚摸或玩耍）
- 家长能通过 Web UI 了解宠物状态并发送有意义的奖励
- 进化路径能反映不同的护理风格（而非单一路径）

| 决策项 | 选择 | 说明 |

| 宠物记忆 | C. 中等记忆 | 存储最近 7 天互动摘要 + 性格数据 |

## Scope

### MVP — 最小可行产品

**核心功能**：
1. **Need 系统**（需求系统）
   - 4 个核心需求：饥饿、快乐、能量、羁绊
   - 每个需求有独立的衰减率和恢复机制
   - 宠物通过视觉表达（气泡、动画）传达需求
   - 需求衰减率设计：
     - 饥饿：1 点/10 分钟（最快）
     - 快乐：1 点/30 分钟
     - 能量：1 点/20 分钟（活跃）/ +2 点/分钟（睡眠）
     - 羁绊：-1 点/2 小时（无交互）

2. **Personality 系统**（性格系统）
   - 6 种性格特质：playful, shy, curious, lazy, brave, gentle
   - 性格通过互动模式发展
   - 影响宠物行为（移动速度、探索频率、反应方式）

3. **增强状态机**
   - 10 种状态：idle, hungry, happy, eating, playing, sleeping, distressed, curious, evolving, branching
   - 状态转换基于需求值和性格
   - 每个状态有对应的视觉和音频反馈

4. **分支进化系统**
   - 5 阶段进化：Egg → Baby → Teen → Adult → Final
   - 进化控制：积分驱动（非时间）
     - 家长给积分 → 孩子买食物 → 喂宠物 → 累积喂食值 → 达到阈值后使用进化石 → 进化
   - 进化阈值（喂食值）：
     - Egg → Baby：100 喂食值 + 进化石
     - Baby → Teen：200 喂食值 + 进化石
     - Teen → Adult：350 喂食值 + 进化石
     - Adult → Final：500 喂食值 + 进化石
   - Teen 阶段根据护理风格分支：
     - Kind（高羁绊 + 均衡护理）→ Guardian Dragon
     - Playful（高快乐 + 多玩耍）→ Trickster Dragon
     - Wild（低羁绊 + 高能量）→ Wild Dragon
     - Wise（规律喂食 + 稳定护理）→ Wise Dragon

5. **交互系统**
   - 按钮 A 短按：喂食（消耗食物）
   - 按钮 A 长按：抚摸（增加羁绊）
   - 按钮 B 短按：玩耍（消耗能量，增加快乐）
   - 按钮 B 长按：商店（购买食物）
   - 按钮 C 短按：休息（恢复能量）
   - 按钮 C 长按：设置（重置）

6. **视觉反馈**
   - 需求气泡（食物、爱心、zzz、思念）
   - 表情动画（开心、饥饿、困倦、孤独）
   - 进化动画序列
   - 基础 UI 叠加层（需求条）

7. **经济系统**
   - 积分获取：家长手动发放 + 任务自动奖励（混合模式）
   - 食物商店（5 种）：
     - 苹果：3 积分，+10 喂食值
     - 鸡腿：5 积分，+20 喂食值
     - 蛋糕：8 积分，+30 喂食值
     - 星星：12 积分，+50 喂食值
     - 进化石：20 积分，触发进化（需要喂食值达标）
   - 互动用途明确：喂食→恢复饥饿，抚摸→增加羁绊，玩耍→增加快乐
   - 积分上限：9999

8. **成就 + 称号系统**
   - **连续喂食成就**（培养责任感）：
     - 连续 3 天喂食 → 积分 +10 + 称号"小饲养员"
     - 连续 7 天喂食 → 积分 +30 + 称号"忠实守护者"
     - 连续 14 天喂食 → 积分 +50 + 称号"喂食大师"
     - 连续 30 天喂食 → 积分 +100 + 称号"永恒守护者"
   - **进化里程碑**（成就感）：
     - 进化到 Baby → 称号"新生命"
     - 进化到 Teen → 称号"成长者"
     - 进化到 Adult → 称号"成熟体"
     - 进化到 Final → 称号"完全体"
   - **情感连接**（鼓励陪伴）：
     - 羁绊值达到 50 → 称号"朋友"
     - 羁绊值达到 80 → 称号"挚友"
     - 羁绊值达到 100 → 称号"灵魂伙伴"
   - **护理大师**（高级目标）：
     - 所有需求同时 ≥ 80 持续 24 小时 → 积分 +50 + 称号"护理大师"

9. **记忆系统**（中等记忆）
   - 存储最近 7 天的互动摘要（每天：喂食次数、抚摸次数、玩耍次数）
   - 性格发展数据（6 维度，0-100）
   - 当前需求值和状态
   - 进化历史和路径
   - 成就列表

10. **Web UI 基础版**
    - 实时宠物状态显示（Canvas 渲染）
    - 需求仪表盘（4 个需求条）
    - 进化进度可视化
    - 远程互动按钮（喂食、抚摸、玩耍）
    - 奖励发送功能（PIN 保护）
    - 护理历史查看
    - 成就展示
    - 远程互动限制：每天最多 10 次喂食/抚摸/玩耍，5 次奖励

11. **疲劳机制**（可视化疲劳）
    - 连续互动 5 次后，宠物显示困倦动画（zzz 气泡）
    - 宠物仍然可以互动，但快乐值不再增加
    - 等待 10 分钟后，宠物恢复活力
    - 教育意义：教孩子"适可而止"，不惩罚过度互动

### Out of Scope — 不在范围内

1. **多物种支持** — 先只做 Dragon，验证核心机制后再扩展
2. **天气系统** — 可选功能，后续添加
3. **背景音乐** — 可选功能，后续添加
4. **OTA 更新** — 保持现有实现
5. **移动端 App** — 保持 Web 界面
6. **宠物死亡/永久消失** — 对儿童情感伤害太大，不实现
7. **复杂社交功能** — 保持简单，避免分心
8. **多宠物同时养** — 保持专注，一次只养一只

## Risks

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| 内存不足 | Medium | High | 优化精灵格式，使用索引颜色，限制粒子数量 |
| CPU 性能不足 | Medium | Medium | 优化渲染循环，使用定时器中断，减少复杂计算 |
| 儿童无法理解需求系统 | Low | High | 设计直观的视觉表达（气泡、颜色），简化需求描述 |
| 进化分支太复杂 | Medium | Medium | 简化路径计算，使用更直观的分类（快乐/生气/困倦/好奇） |
| 家长不愿使用 Web UI | High | Medium | 简化界面，提供快速操作（一键奖励），移动友好 |
| 硬件按钮损坏 | Low | High | 提供 Web 界面作为备用交互方式 |
| WiFi 不稳定 | Medium | Low | 离线优先设计，本地缓存，断线重连 |
| 积分经济失衡 | Medium | Medium | 调整食物价格和喂食值，保持游戏节奏 |
| 成就系统过于复杂 | Low | Medium | 保持成就数量适中，每个成就有明确意义 |
